══════════════════════════════════════════════════════════════
PARTE 0 — O QUE É O deepH Plus?
══════════════════════════════════════════════════════════════

O deepH Plus é o seu ESCRITÓRIO PESSOAL DE ASSISTENTES DE IA,
instalado diretamente no seu computador. Tudo roda local,
sem enviar seus dados para nenhum servidor externo.

Pense assim: imagine que você contratou vários "funcionários
virtuais especializados". Cada um tem nome, personalidade
e habilidades definidas por você.

Você pode ter, por exemplo:
  - Um "Consultor Financeiro" para análise de investimentos
  - Um "Redator" para escrever textos e e-mails
  - Um "Programador" para ajudar com código
  - Um "Pesquisador" para buscar informações

Cada um desses é chamado de AGENTE.

O deepH Plus é o sistema que gerencia esses agentes através
de uma INTERFACE VISUAL MODERNA E PREMIUM, com:
  ✓ Tema escuro elegante (Deep Black + Emerald Green)
  ✓ Animações suaves e design premium
  ✓ Chat em tempo real com formatação rica (Markdown)
  ✓ Painel de controle visual intuitivo
  ✓ Suporte a múltiplos provedores de IA (DeepSeek, OpenAI,
    Anthropic, Groq, Ollama local e mais)

[*] Todas as conversas e configurações ficam no SEU
    computador. Nada vai para nuvem exceto a mensagem
    que você envia ao modelo de IA escolhido.

PARTE 1 — INSTALAÇÃO E PRÉ-REQUISITOS
══════════════════════════════════════════════════════════════

Antes de abrir o deepH Plus, certifique-se de que você tem:

  1. Windows 10 ou 11 (64-bit)
  2. O arquivo deeph.exe na pasta C:\\Users\\BOSS\\deepH
  3. Conexão com internet (para chamar a IA via API)

Você NÃO precisa instalar nada especial. O deepH Plus é
um arquivo único executável (.exe) em Go, que já carrega
o painel web dentro de si mesmo.

──────────────────────────────────────────────────────────────
Estrutura de Pastas do deepH Plus
──────────────────────────────────────────────────────────────

Sua pasta C:\\Users\\BOSS\\deepH contém:

  deeph.exe        → O programa principal (motor em Go)
  start_ui.bat     → Atalho de duplo clique para iniciar
  config.json      → Suas configurações e API Keys (gerado
                     automaticamente na 1ª execução)
  agents/          → Pasta com seus agentes (.yaml)
  skills/          → Skills instaladas (habilidades extras)
  sessions/        → Histórico de conversas salvo

[!] NUNCA apague o arquivo config.json sem querer!
    Ele contém suas chaves de API. Faça backup dele.

PARTE 4 — CONFIGURANDO SUAS CHAVES DE API
══════════════════════════════════════════════════════════════

Para o deepH Plus chamar a inteligência artificial, você
precisa de uma CHAVE DE API do provedor de IA escolhido.

Pense na chave de API como uma "senha de acesso" ao cérebro
da IA. Sem ela, o motor não consegue gerar respostas.

──────────────────────────────────────────────────────────────
4.1 — Onde Obter sua Chave (Gratuito ou Pago)
──────────────────────────────────────────────────────────────

Principais provedores e onde criar conta:

  DEEPSEEK (Recomendado — muito barato!)
  → Site: platform.deepseek.com
  → Modelo principal: deepseek-chat
  → Custo: centavos por milhão de tokens (~R$0,01/resposta)
  → Chave começa com: sk-...

  OPENAI
  → Site: platform.openai.com
  → Modelos: gpt-4o, gpt-4o-mini
  → Custo: moderado
  → Chave começa com: sk-proj-...

  ANTHROPIC (CLAUDE)
  → Site: console.anthropic.com
  → Modelo: claude-3-5-sonnet
  → Custo: moderado/alto
  → Chave começa com: sk-ant-...

  GROQ (Ultra rápido, gratuito até certo limite!)
  → Site: console.groq.com
  → Modelos: llama-3.3-70b, mixtral
  → Chave começa com: gsk_...

  OLLAMA (100% GRATUITO — roda no seu PC!)
  → Site: ollama.ai
  → Sem internet, sem custo, sem chave!
  → Requer GPU ou processador potente

──────────────────────────────────────────────────────────────
4.2 — Inserindo a Chave no deepH Plus
──────────────────────────────────────────────────────────────

[>] Passo 1: Clique em "Configurações" (ícone ⚙️) na barra
             lateral esquerda

[>] Passo 2: Você verá a tela "System Configuration" com
             seções para cada provedor

[>] Passo 3: Encontre o provedor desejado (ex: DeepSeek)
             e clique no campo de texto escuro

[>] Passo 4: COLE sua chave de API no campo
             (use CTRL+V para colar)

[>] Passo 5: Clique no botão "Save Settings" (verde neon)
             no final da página

[!] SEGURANÇA: A chave fica salva APENAS no arquivo
    config.json no seu computador. Ela só é enviada
    DIRETAMENTE ao servidor da IA quando você envia
    uma mensagem. Nunca passamos por servidores nossos.

[*] Você pode configurar VÁRIOS provedores ao mesmo tempo!
    Cada agente usa o provedor definido no seu arquivo .yaml.

──────────────────────────────────────────────────────────────
4.3 — Verificando se a Configuração Funcionou
──────────────────────────────────────────────────────────────

Após salvar, as chaves configuradas aparecerão na tela de
configurações com status atingido. Tente conversar com
um agente para confirmar que a resposta da IA chegou.

Se der erro, verifique:
  ✗ A chave foi copiada completa? (sem espaços extras)
  ✗ A conta tem créditos disponíveis?
  ✗ O provedor no agente e na config são o mesmo?

PARTE 5 — O DASHBOARD: SEUS AGENTES
══════════════════════════════════════════════════════════════

O DASHBOARD é a tela inicial — é onde vivem seus agentes.

──────────────────────────────────────────────────────────────
5.1 — Entendendo os Cartões de Agente
──────────────────────────────────────────────────────────────

Cada agente aparece como um CARTÃO TRANSLÚCIDO na tela.
Um cartão contém:

  ┌──────────────────────────────────────┐
  │ 🤖  NOME DO AGENTE         [Status] │
  │     Provedor: deepseek              │
  │                                      │
  │  "Descrição do que este agente faz  │
  │   e quais habilidades tem."          │
  │                                      │
  │  [CONVERSAR]        [EDITAR] ⚙️     │
  └──────────────────────────────────────┘

  • NOME: O nome personalizado que você deu ao agente
  • STATUS: Indica se está ativo (ponto verde pulsante)
  • PROVEDOR: Qual IA ele usa (DeepSeek, OpenAI etc)
  • DESCRIÇÃO: O que ele faz (você que define isso!)
  • CONVERSAR: Abre o chat com este agente
  • EDITAR: Abre o editor para modificá-lo

──────────────────────────────────────────────────────────────
5.2 — O Agente Padrão "guide"
──────────────────────────────────────────────────────────────

Por padrão, o deepH Plus vem com um agente chamado "guide".
Ele é configurado para DeepSeek e serve como assistente geral.

Você pode:
  → Conversar com ele já (se tiver chave DeepSeek)
  → Editá-lo para mudar o nome, provedor ou especialidade
  → Usá-lo como modelo para criar novos agentes

──────────────────────────────────────────────────────────────
5.3 — Criando um Novo Agente pelo Painel
──────────────────────────────────────────────────────────────

[>] Clique no botão "Novo Agente" (ícone ➕) na barra lateral
[>] A tela de criação de agente vai abrir
[>] Preencha os campos (veja a Parte 7 para detalhes)
[>] Clique em "Deploy Agent" (salvar)

O novo agente aparecerá no Dashboard imediatamente!

──────────────────────────────────────────────────────────────
6.4 — Formatação Rica nas Respostas
──────────────────────────────────────────────────────────────

O Chat suporta MARKDOWN completo nas respostas da IA:

  • **Texto em negrito**
  • *Texto em itálico*
  • # Títulos e subtítulos
  • Listas com marcadores
  • Blocos de código com highlight
  • Tabelas formatadas
  • Links clicáveis

[*] Peça para o agente formatar a resposta em markdown
    para respostas mais organizadas e legíveis!

──────────────────────────────────────────────────────────────
6.5 — Ações nas Mensagens
──────────────────────────────────────────────────────────────

Na última mensagem da IA, você verá dois botões:

  [Copiar] → Copia todo o texto da resposta para o clipboard
  [Regenerar] → Pede para a IA gerar uma resposta diferente

[*] Use Regenerar se a resposta não foi satisfatória!
    A IA vai tentar de novo com uma abordagem diferente.

──────────────────────────────────────────────────────────────
6.6 — O Chat é Persistente!
──────────────────────────────────────────────────────────────

Cada conversa é salva automaticamente em:
  C:\\Users\\BOSS\\deepH\\sessions\\

Ficam salvas em arquivos .json com o histórico completo.
Isso significa que a IA consegue "lembrar" das últimas
mensagens da conversa para manter o contexto.

[!] O histórico é enviado a cada nova mensagem para manter
    o contexto. Conversas muito longas aumentam o custo por
    chamada. Em caso de problemas, reinicie o chat abrindo
    o agente novamente (uma nova sessão começa).

PARTE 7 — CRIANDO E EDITANDO AGENTES
══════════════════════════════════════════════════════════════

O Editor de Agentes é onde você PERSONALIZA e CRIA seus
assistentes virtuais!

──────────────────────────────────────────────────────────────
7.1 — Abrindo o Editor
──────────────────────────────────────────────────────────────

PARA CRIAR UM NOVO AGENTE:
  [>] Clique em "Novo Agente" (➕) na barra lateral

PARA EDITAR UM AGENTE EXISTENTE:
  [>] No Dashboard, clique em "Editar" (⚙️) no cartão dele

──────────────────────────────────────────────────────────────
7.2 — Os Campos do Editor
──────────────────────────────────────────────────────────────

O editor possui os seguintes campos:

  NOME DO AGENTE
  → O apelido do seu agente (ex: "Consultor", "RedatorPro")
  → Use nomes sem espaços ou caracteres especiais
  → Exemplo: consultor_financeiro

  DESCRIÇÃO
  → O que este agente faz? Explique em 1-2 frases.
  → Isso aparece no cartão do Dashboard
  → Exemplo: "Analisa investimentos e mercado financeiro"

  PROVEDOR (Provider)
  → Qual empresa de IA vai "dar vida" a este agente?
  → Opções: deepseek, openai, anthropic, groq, ollama
  → Escolha o mesmo provedor cuja chave você configurou

  MODELO (Model)
  → O nome específico do modelo dentro do provedor
  → DeepSeek: deepseek-chat, deepseek-reasoner
  → OpenAI: gpt-4o, gpt-4o-mini, gpt-3.5-turbo
  → Anthropic: claude-3-5-sonnet-20241022
  → Groq: llama-3.3-70b-versatile, mixtral-8x7b-32768
  → Ollama: llama3, mistral, phi3

  SYSTEM PROMPT (Regras do Agente)
  → Este é o campo mais importante!
  → São as INSTRUÇÕES que definem a personalidade e
    especialidade do agente
  → Exemplo para um Consultor Financeiro:
    "Você é um consultor financeiro brasileiro experiente.
     Sempre forneça análises equilibradas, mencione riscos
     e benefícios. Fale em português do Brasil. Responda
     de forma objetiva e profissional."

  SKILLS (Habilidades)
  → Skills são capacidades extras que o agente pode usar
  → São arquivos SKILL.md instalados na pasta skills/
  → Selecione as skills relevantes para o agente
  → Elas aparecem como cards verdes quando ativadas

──────────────────────────────────────────────────────────────
7.3 — Boas Práticas para System Prompts
──────────────────────────────────────────────────────────────

Um bom System Prompt deve conter:

  1. PERSONA: Quem ele é?
     "Você é um especialista em marketing digital..."

  2. COMPORTAMENTO: Como ele deve agir?
     "Seja conciso, direto e use exemplos práticos..."

  3. IDIOMA: Em que língua responde?
     "Responda sempre em português do Brasil..."

  4. RESTRIÇÕES: O que ele NÃO deve fazer?
     "Não dê conselhos médicos. Não discuta política..."

  5. FORMATO: Como formatar as respostas?
     "Use listas quando listar itens. Use negrito para
      pontos importantes. Limite respostas a 300 palavras..."

──────────────────────────────────────────────────────────────
7.4 — Salvando o Agente
──────────────────────────────────────────────────────────────

[>] Após preencher todos os campos, clique em "Deploy Agent"

O agente será salvo como arquivo .yaml em:
  C:\\Users\\BOSS\\deepH\\agents\\[nome-do-agente].yaml

[*] Você pode editar este arquivo .yaml diretamente com
    um editor de texto como o Notepad++ para ajustes rápidos!

──────────────────────────────────────────────────────────────
7.5 — Exemplo de Arquivo YAML de Agente
──────────────────────────────────────────────────────────────

Este é o que o deepH Plus gera internamente:

  name: consultor_financeiro
  description: Analista de investimentos e mercado
  provider: deepseek
  model: deepseek-chat
  system_prompt: |
    Você é um consultor financeiro brasileiro experiente.
    Analise investimentos com foco em risco e retorno.
    Sempre mencione as variáveis do mercado brasileiro.
    Responda de forma clara e objetiva em português.
  skills:
    - web_search
    - memory

PARTE 8 — USANDO NO TERMINAL (MODO AVANÇADO)
══════════════════════════════════════════════════════════════

O deepH Plus também funciona 100% pelo terminal, sem precisar
abrir o navegador. Isso é útil para automações e scripts.

──────────────────────────────────────────────────────────────
8.1 — Comandos Principais
──────────────────────────────────────────────────────────────

Abra o PowerShell na pasta C:\\Users\\BOSS\\deepH e use:

  INICIAR A INTERFACE VISUAL:
  .\\deeph.exe ui

  RODAR UM AGENTE DIRETAMENTE:
  .\\deeph.exe run "guide" "Qual a capital do Brasil?"

  VER A LISTA DE AGENTES:
  .\\deeph.exe agents list

  VER AS SKILLS INSTALADAS:
  .\\deeph.exe skills list

  RODAR MÚLTIPLOS AGENTES (Orquestração):
  .\\deeph.exe run "agente1 + agente2" "Sua tarefa aqui"

──────────────────────────────────────────────────────────────
8.2 — Exemplo de Uso no Terminal
──────────────────────────────────────────────────────────────

[>] Abra o PowerShell (Win+X → Terminal do Windows)
[>] Navegue até a pasta:
    cd C:\\Users\\BOSS\\deepH
[>] Execute:
    .\\deeph.exe run "guide" "Me explique o que é inflação"

Você verá a resposta completa aparecer diretamente no terminal!

[*] Para scripts automáticos e pipelines, o modo terminal
    é o mais poderoso. Pode redirecionar a saída para
    arquivos: .\\deeph.exe run "guide" "tarefa" > resposta.txt

──────────────────────────────────────────────────────────────

  SE VOCÊ QUER ECONOMIZAR E TER QUALIDADE:

══════════════════════════════════════════════════════════════
PARTE 11 — VÁRIOS AGENTES TRABALHANDO JUNTOS
══════════════════════════════════════════════════════════════

Uma das capacidades mais poderosas do deepH Plus é a
ORQUESTRAÇÃO DE MÚLTIPLOS AGENTES. Você pode fazer
vários agentes trabalharem em equipe — ao mesmo tempo
ou em sequência — para resolver tarefas complexas.

Pense nisso como uma equipe de funcionários: cada um
traz uma especialidade, e o resultado é muito melhor
do que qualquer um sozinho.

──────────────────────────────────────────────────────────────
11.1 — SINTAXE DE ORQUESTRAÇÃO
──────────────────────────────────────────────────────────────

O controle é feito pelo comando run no terminal, com
dois operadores especiais:

  O SINAL DE + (mais) → significa PARALELO
  Os agentes rodam ao MESMO TEMPO, de forma independente.

  O SINAL DE > (maior que) → significa SEQUÊNCIA
  O resultado do primeiro é passado como entrada ao próximo.

──────────────────────────────────────────────────────────────
11.2 — MODO PARALELO (+ )
──────────────────────────────────────────────────────────────

Uso: dois ou mais agentes recebem a mesma tarefa e
trabalhando SIMULTANEAMENTE. O sistema aguarda que
ambos terminem antes de prosseguir.

EXEMPLO:
  .\deeph.exe run "analista+redator" "Pesquise sobre inflação"

O que acontece:
  → O agente "analista" roda ao mesmo tempo que o "redator"
  → Ambos recebem a mesma tarefa
  → Os resultados são exibidos lado a lado

[*] Útil quando você quer comparar perspectivas diferentes
    ou dividir um trabalho em partes independentes.

──────────────────────────────────────────────────────────────
11.3 — MODO SEQUENCIAL (>)
──────────────────────────────────────────────────────────────

Uso: o OUTPUT do primeiro agente vira o INPUT do
proximo. É como uma linha de montagem — cada etapa
melhora o trabalho da anterior.

EXEMPLO:
  .\deeph.exe run "planejador>codificador>revisor" "Crie um script"

O que acontece:
  1. O "planejador" recebe a tarefa e cria um plano
  2. O "codificador" recebe o plano e escreve o código
  3. O "revisor" recebe o código e faz a revisão final

[*] Ideal para pipelines de trabalho com etapas bem definidas.
    O resultado de cada agente alimenta o próximo!

──────────────────────────────────────────────────────────────
11.4 — MODO COMBINADO (paralelo + sequencial)
──────────────────────────────────────────────────────────────

Você pode MISTURAR os dois modos na mesma execução!

EXEMPLO:
  .\deeph.exe run "planejador+leitor>codificador>revisor" "Tarefa"

O que acontece:
  1. "planejador" e "leitor" rodam AO MESMO TEMPO
  2. O "codificador" recebe os DOIS resultados e os integra
  3. O "revisor" faz a revisão final do trabalho combinado

[!] Este é o modo mais poderoso. Use para tarefas complexas
    que exigem coleta de informações paralelas e depois
    uma síntese ou produção final.

──────────────────────────────────────────────────────────────
11.5 — SYNTAX COMPLETA DO COMANDO run
──────────────────────────────────────────────────────────────

  .\deeph.exe run [opções] "<spec>" "[sua tarefa aqui]"

OPÇÕES DISPONÍVEIS:
  --trace          Exibe o plano de execução detalhado
  --coach=false    Desativa as dicas automáticas do Coach
  --multiverse N   Roda N universos paralelos para comparar
                   resultados de uma mesma equipe de agentes
  --judge-agent X  Usa o agente X para escolher o
                   melhor resultado entre os universos

EXEMPLOS COMPLETOS:
  [>] Agente simples:
      .\deeph.exe run "guide" "Me fale sobre Go lang"

  [>] Dois agentes em paralelo:
      .\deeph.exe run "pesquisador+redator" "Escreva sobre IA"

  [>] Pipeline de três etapas:
      .\deeph.exe run "planejador>dev>revisor" "Crie uma API REST"

  [>] Ver o plano antes de executar (trace):
      .\deeph.exe trace "planejador+leitor>revisor" "Minha tarefa"

[*] O comando trace NÃO executa os agentes — ele apenas
    mostra como a tarefa seria dividida entre eles.
    Ótimo para entender o fluxo antes de rodar de verdade!

──────────────────────────────────────────────────────────────
11.6 — USANDO CREWS (Times Pré-Definidos)
──────────────────────────────────────────────────────────────

Além de compor equipes manualmente, você pode criar
arquivos .yaml de "crew" na pasta crews/ para salvar
configurações de times reutilizáveis.

COMANDOS DE CREW:
  .\deeph.exe crew list           → Lista os crews disponíveis
  .\deeph.exe crew show <nome>    → Detalhes de um crew
  .\deeph.exe run "@reviewpack"   → Roda o crew "reviewpack"

[*] Crews são úteis para equipes que você usa repetidamente.
    Crie uma vez e reutilize com facilidade!

PARTE 12 — USANDO KITS PRONTOS
══════════════════════════════════════════════════════════════

O deepH Plus oferece KITS — pacotes prontos que instalam
automaticamente um conjunto de agentes, skills e configurações
para um caso de uso específico.

Pense nos kits como "templates de projeto" — você instala
um kit e já tem tudo configurado para começar a trabalhar!

──────────────────────────────────────────────────────────────
12.1 — VER KITS DISPONÍVEIS
──────────────────────────────────────────────────────────────

[>] Abra o PowerShell na pasta C:\Users\BOSS\deepH
[>] Execute:
    .\deeph.exe kit list

A lista de kits disponíveis vai aparecer no terminal
com nome, descrição, número de skills e arquivos incluídos:

  - hello-next-tailwind: Starter kit para Next.js com Tailwind
    (skills=2 files=3 provider=deepseek)
  - reviewpack: Kit de revisão de código com múltiplos agentes
    (skills=3 files=2 provider=deepseek)
  ...

──────────────────────────────────────────────────────────────
12.2 — INSTALAR UM KIT
──────────────────────────────────────────────────────────────

[>] Para instalar um kit, execute:
    .\deeph.exe kit add <nome-do-kit>

EXEMPLO:
  .\deeph.exe kit add hello-next-tailwind

O que acontece automaticamente:
  ✓ As skills necessárias são instaladas na pasta skills/
  ✓ Os arquivos de agentes .yaml são criados na pasta agents/
  ✓ Um provider DeepSeek é configurado (se necessário)
  ✓ A validação é feita e o kit está pronto para uso!

[!] IMPORTANTE: Execute deeph init antes de instalar kits!
    O workspace precisa estar inicializado primeiro.
    Se ainda não fez isso: .\deeph.exe init

──────────────────────────────────────────────────────────────
12.3 — OPÇÕES DO COMANDO kit add
──────────────────────────────────────────────────────────────

  .\deeph.exe kit add [opções] <nome-do-kit>

OPÇÕES DISPONÍVEIS:
  --force              Sobrescreve arquivos existentes
  --provider-name X    Nome do provider a criar (padrão: deepseek)
  --model X            Modelo a usar (padrão: deepseek-chat)
  --set-default-provider  Define o provider instalado como padrão
  --skip-provider      Não configura nenhum provider

EXEMPLOS:
  [>] Instalação simples:
      .\deeph.exe kit add reviewpack

  [>] Forçar reinstalação (sobrescrever arquivos):
      .\deeph.exe kit add reviewpack --force

  [>] Usar provider e modelo específicos:
      .\deeph.exe kit add reviewpack --provider-name meu-ds --model deepseek-reasoner

──────────────────────────────────────────────────────────────
12.4 — APÓS INSTALAR UM KIT
──────────────────────────────────────────────────────────────

Após instalar, o terminal mostrará um resumo e os
próximos passos recomendados:

  Installed kit "reviewpack" in C:\Users\BOSS\deepH
    source: built-in
    skills:  created=2 updated=0 unchanged=0 skipped=0
    files:   created=1 updated=0 unchanged=0 skipped=0
    provider: added deepseek provider "deepseek" + default_provider
  Next steps:
    1. deeph validate
    2. deeph crew list
    3. deeph run --multiverse 0 @reviewpack "sua tarefa"

[>] Execute os passos sugeridos em ordem:
    1. .\deeph.exe validate
    2. .\deeph.exe crew list
    3. .\deeph.exe run "@reviewpack" "Revise este código"

[*] Kits que incluem crews podem ser executados com a
    sintaxe @nome-do-crew para rodar todos os agentes
    do kit de uma só vez!


==============================================================
GLOSSÁRIO — PALAVRAS QUE VOCÊ PRECISA SABER
==============================================================

  AGENTE: Um assistente virtual com personalidade e
          especialidade definidas por você.

  API KEY: Senha de acesso ao serviço de IA na nuvem.
           Obtida no site do provedor (OpenAI, DeepSeek etc)

  BACKEND: O "motor" do sistema. É a janela preta do
           terminal que roda o servidor em Go.

  FRONTEND: A interface visual que você vê no navegador.

  LOCALHOST: Endereço do servidor rodando no seu próprio
             computador (não vai para internet).

  MODELO: O "cérebro" específico da IA (ex: GPT-4o é um
          modelo da OpenAI, deepseek-chat é do DeepSeek).

  PROVEDOR: A empresa que criou e hospeda o modelo de IA.

  CREW: Um time pré-definido de agentes salvo como arquivo
        .yaml na pasta crews/. Pode ser acionado com @crew.

  KIT: Pacote de instalação pronto com agentes, skills e
       configurações para um caso de uso específico.

  ORQUESTRAÇÃO: Coordenação de múltiplos agentes usando
                + (paralelo) e > (sequencial) para criar
                fluxos de trabalho complexos.

  SKILL: Uma habilidade extra que você pode adicionar
         a um agente (ex: pesquisa web, memória, código).

  SSE: Server-Sent Events — tecnologia que permite o
       texto aparecer em tempo real (palavra por palavra).

  SYSTEM PROMPT: As instruções secretas que definem a
                 personalidade e comportamento do agente.

  TOKEN: Unidade de cobrança das APIs de IA. Uma palavra
         = ~1,3 tokens. Cada chamada consome tokens.

  YAML: Formato de arquivo de configuração usado pelos
        agentes. Fácil de ler e editar com qualquer editor.

PARTE 0 — O QUE É O deepH Plus?
══════════════════════════════════════════════════════════════

O deepH Plus é o seu ESCRITÓRIO PESSOAL DE ASSISTENTES DE IA,
instalado diretamente no seu computador. Tudo roda local,
sem enviar seus dados para nenhum servidor externo.

Pense assim: imagine que você contratou vários "funcionários
virtais especializados". Cada um tem nome, personalidade
e habilidades definidas por você.

Você pode ter, por exemplo:
  - Um "Consultor Financeiro" para análise de investimentos
  - Um "Redator" para escrever textos e e-mails
  - Um "Programador" para ajudar com código
  - Um "Pesquisador" para buscar informações

Cada um desses é chamado de AGENTE.

O deepH Plus é o sistema que gerencia esses agentes através
de uma INTERFACE VISUAL MODERNA E PREMIUM, com:
  ✓ Tema escuro elegante (Deep Black + Emerald Green)
  ✓ Animações suaves e design premium
  ✓ Chat em tempo real com formatação rica (Markdown)
  ✓ Painel de controle visual intuitivo
  ✓ Suporte a múltiplos provedores de IA (DeepSeek, OpenAI,
    Anthropic, Groq, Ollama local e mais)

[*] Todas as conversas e configurações ficam no SEU
    computador. Nada vai para nuvem exceto a mensagem
    que você envia ao modelo de IA escolhido.

──────────────────────────────────────────────────────────────
3.1 — A BARRA LATERAL (Menu de Navegação)
──────────────────────────────────────────────────────────────

À esquerda ficam os botões de navegação. Cada um tem um
ícone e texto:

  🏠 Dashboard    → Gerencie seus agentes individuais
  👥 Esquadrões   → Orchestre múltiplos agentes em equipe
  💬 Chat         → Janela de conversa inteligente
  ➕ Novo Agente  → Criar um novo agente do zero
  ⚙️ Configurações → Gerenciar API Keys e preferências

[*] As transições entre as telas são suaves e animadas!


══════════════════════════════════════════════════════════════
PARTE 4 — CONFIGURANDO SUAS CHAVES DE API
══════════════════════════════════════════════════════════════

Para o deepH Plus chamar a inteligência artificial, você
precisa de uma CHAVE DE API do provedor de IA escolhido.

[>] Passo 1: Clique em "Configurações" (ícone ⚙️) na sidebar.
[>] Passo 2: Encontre o provedor (ex: DeepSeek).
[>] Passo 3: COLE sua chave de API e clique em "Save Settings".

[!] SEGURANÇA: A chave fica salva APENAS no seu computador.
    Nunca passamos seus segredos por nossos servidores.


══════════════════════════════════════════════════════════════
PARTE 5 — O DASHBOARD E AGENTES
══════════════════════════════════════════════════════════════

O DASHBOARD mostra seus agentes como cartões translúcidos.
Cada agente tem uma especialidade e um "System Prompt" (instruções).

  • CONVERSAR: Abre o chat direto.
  • EDITAR: Altera a personalidade ou provedor (modelo).
  • EXCLUIR: Remove o agente definitivamente do sistema.


══════════════════════════════════════════════════════════════
PARTE 6 — CONVERSANDO NO CHAT
══════════════════════════════════════════════════════════════

O chat do deepH Plus v4.0 agora é mais inteligente! 
Ele utiliza o **ContextBus** para lembrar apenas do que é 
importante, economizando seus tokens de API.

  • SUAS MENSAGENS: Lado direito, fundo verde neon.
  • RESPOSTAS IA: Lado esquerdo, estilo vidro (Glassmorphism).

[*] O chat suporta Markdown completo (negrito, tabelas, código).


══════════════════════════════════════════════════════════════
PARTE 7 — CRIANDO AGENTES MESTRES
══════════════════════════════════════════════════════════════

No editor, o campo mais vital é o **SYSTEM PROMPT**.
Define quem a IA é e como deve reagir.

Exemplo de Redator:
  "Você é um redator sênior de marketing. Escreva posts
   persuasivos para Instagram. Sempre termine com uma
   pergunta para engajar a audiência."


══════════════════════════════════════════════════════════════
PARTE 8 — ESQUADRÕES (CREWS): O PODER DA EQUIPE
══════════════════════════════════════════════════════════════

A grande novidade da v4.0 são os **ESQUADRÕES**.
Em vez de falar com um agente, você fala com uma EQUIPE.

[>] EXEMPLO: Crew "build-calculator"
Este esquadrão contém 3 agentes trabalhando em sequência:
  1. Arquiteto: Planeja as funcionalidades.
  2. Desenvolvedor: Escreve o código HTML/CSS/JS.
  3. Assistente de Implantação: Abre a janela de salvar arquivo.

Para rodar este exemplo no terminal:
  .\deeph.exe run "build-calculator" "Crie uma calculadora neon"

[*] Esquadrões usam conectores especiais:
  "+" → Agentes rodam ao mesmo tempo.
  ">" → Um agente passa o bastão para o próximo.


══════════════════════════════════════════════════════════════
PARTE 9 — O COACH: DIAGNÓSTICO EM TEMPO REAL
══════════════════════════════════════════════════════════════

O deepH Plus agora vem com o **COACH**.
Ele monitora as execuções e dá dicas se algo não estiver 100%.

  • Ver estatísticas: .\deeph.exe coach stats
  • Resetar dicas: .\deeph.exe coach reset --all

O Coach ajuda você a otimizar seus prompts e economizar tokens.
